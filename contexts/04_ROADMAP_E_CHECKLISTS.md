Roadmap de Desenvolvimento e ChecklistsO desenvolvimento será dividido em Sprints de 2 semanas, com um Sprint 0 inicial de 1 semana.Sprint 0: Configuração e Setup (1 semana)[ ] Definir e configurar repositório Git (ex: GitHub).[ ] Configurar projeto backend (NestJS) com estrutura inicial de módulos.[ ] Configurar projeto frontend (React + Vite) com estrutura de pastas.[ ] Instalar dependências principais em ambos os projetos.[ ] Configurar Docker Compose com imagem do PostgreSQL.[ ] Estabelecer pipeline de CI/CD inicial no Vercel e Render.[ ] Realizar um deploy "Hello World" de ambas as aplicações.Sprint 1: Core Backend e Autenticação (2 semanas)[ ] Finalizar o schema do Prisma (User, DiaryEntry).[ ] Gerar e executar a migração inicial do banco de dados.[ ] Implementar módulo de Users (CRUD básico).[ ] Implementar módulo de Auth com Passport.js e JWT.[ ] Criar endpoint /auth/register (com hash de senha).[ ] Criar endpoint /auth/login (com Access/Refresh Tokens).[ ] Implementar Guards do NestJS para proteger rotas.[ ] Escrever testes unitários para os serviços de autenticação.Sprint 2: Frontend Básico e Conexão (2 semanas)[ ] Desenvolver o componente de Layout principal (Navegação, etc.).[ ] Criar a página de Cadastro e seu formulário.[ ] Criar a página de Login e seu formulário.[ ] Implementar armazenamento seguro do JWT no cliente.[ ] Configurar o roteamento com React Router (rotas públicas e privadas).[ ] Criar a estrutura da página de Dashboard (sem dados).[ ] Implementar a funcionalidade de Logout.Sprint 3: Funcionalidade Principal (Diário) (2 semanas)[ ] Implementar o módulo Diary no backend (CRUD completo).[ ] Garantir que os endpoints do diário sejam protegidos.[ ] Desenvolver o formulário completo de "Novo Registro" no frontend.[ ] Implementar a lógica de submissão do formulário via API.[ ] Desenvolver a visualização de um registro único.[ ] Implementar a funcionalidade de edição de um registro.[ ] Popular a Dashboard com a lista dos registros do usuário.Sprint 4: Evolução e PWA (2 semanas)[ ] Criar endpoints de agregação de dados para revisões no backend.[ ] Desenvolver a página de "Revisão Semanal" no frontend.[ ] Desenvolver a página de "Revisão Mensal" com um gráfico simples.[ ] Configurar o arquivo manifest.json.[ ] Implementar e registrar o Service Worker para cache.[ ] Implementar lógica de cache para dados da API.[ ] Testar a funcionalidade "Adicionar à Tela de Início".[ ] Testar o funcionamento offline e sincronização.Sprint 5: Polimento e Lançamento (2 semanas)[ ] Realizar testes de ponta a ponta (E2E) com Cypress ou Playwright.[ ] Revisar e garantir a responsividade da interface.[ ] Otimizar o desempenho da aplicação (Lighthouse).[ ] Realizar uma rodada final de testes de usabilidade.[ ] Configurar monitoramento de erros (Sentry).[ ] Fazer o deploy final para o ambiente de produção.[ ] Monitorar ativamente os logs e o desempenho pós-lançamento.
