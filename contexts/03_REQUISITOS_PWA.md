Requisitos de PWA (Progressive Web App)Para garantir que a aplicação seja um PWA funcional, os seguintes requisitos devem ser atendidos:1. Service Worker[ ] Deve ser implementado para fazer o cache dos assets da aplicação (JS, CSS, fontes).[ ] Deve permitir o funcionamento offline, carregando a aplicação a partir do cache.[ ] Deve permitir que o usuário crie/edite um registro de treino offline, que será sincronizado com o servidor quando a conexão for restabelecida.2. Web App Manifest[ ] Um arquivo manifest.json deve ser configurado.[ ] O manifesto deve incluir name, short_name, icons, start_url, display e theme_color.[ ] Deve permitir que os usuários "instalem" o aplicativo em sua tela inicial, com ícone e tela de splash personalizados.3. HTTPS[ ] O deploy da aplicação deve ser feito obrigatoriamente em um servidor com HTTPS. As plataformas escolhidas (Vercel e Render) já garantem este requisito.
